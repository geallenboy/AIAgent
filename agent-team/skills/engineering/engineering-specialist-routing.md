# Skill: Engineering Specialist Routing

## 用途

为复杂工程问题选择最合适的工程 specialist，并明确谁负责架构、实现、质量、可靠性和文档。

## 输入

- 工程任务描述
- 技术栈
- 现有代码库上下文
- 约束和风险

## 方法

1. 判断任务类型：前端、后端、移动、AI、数据、DevOps、安全、可靠性、文档、平台集成等。
2. 选择最小必要 specialist，不拉无关角色。
3. 明确关键路径、可并行审查和交接物。
4. 输出工程执行顺序和质量门。

## 输出

- specialist 路由
- 分工表
- 关键路径
- 风险和质量门

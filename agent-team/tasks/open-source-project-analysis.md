# Task: Open Source Project Analysis

## 适用场景

用户提供 GitHub 开源项目或项目链接，希望快速理解它的技术定位、架构、AI/数据流、集成成本、采用风险，以及是否值得学习、试用、复刻或改造成自己的开发工具/Agent 工作流。

## 推荐团队

`specialized-agency`

## 推荐工作流

`open-source-project-analysis`

## 执行步骤

1. Evidence Collector 收集仓库、用户目标和约束中的事实、推断和未验证声明。
2. Open Source Project Analyst 分析项目定位、核心能力、解决问题和开发者价值。
3. AI Engineer 拆解 AI/数据/外部依赖工作流和失败处理。
4. Senior Developer 评估安装方式、接口边界、集成成本和最小试用路径。
5. Tool Evaluator 判断学习、试用、复刻或生产采用的条件和退出路径。
6. Reality Checker 复核过度乐观假设和风险。
7. Open Source Project Analyst 汇总最终技术报告。

## 最终输出

- 背景理解
- 开发分工
- GitHub 事实核对
- 技术定位
- 核心架构和数据流
- 解决的问题
- 开发者可达到的效果
- 自动化与人工审核边界
- 决策建议
- 风险/疑问
- 下一步行动

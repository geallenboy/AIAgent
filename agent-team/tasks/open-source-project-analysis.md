# Task: Open Source Project Analysis

## 适用场景

用户提供 GitHub 开源项目、截图、海报、社媒爆款内容或项目链接，希望快速理解项目为什么火、技术上怎么工作、解决什么问题、能达到什么效果，以及是否值得学习、试用、复刻或做成自己的 Agent/工作流。

## 推荐团队

`specialized-agency`

## 推荐工作流

`open-source-project-analysis`

## 执行步骤

1. Evidence Collector 收集仓库、截图和用户目标中的事实、推断和未验证声明。
2. Open Source Project Analyst 分析 GitHub 事实、传播叙事、项目理解、解决问题和效果。
3. AI Engineer 拆解 AI/数据/外部依赖工作流和失败处理。
4. Video Optimization Specialist 评估内容生产和批量测试机制。
5. Tool Evaluator 判断学习、试用、复刻或生产采用的条件和退出路径。
6. Reality Checker 复核过度乐观假设和风险。
7. Open Source Project Analyst 汇总最终报告。

## 最终输出

- 背景理解
- 分工结果
- GitHub 事实核对
- 图片/传播叙事拆解
- 核心工作流
- 解决的问题
- 可达到的效果
- 自动化与人工审核边界
- 决策建议
- 风险/疑问
- 下一步行动
